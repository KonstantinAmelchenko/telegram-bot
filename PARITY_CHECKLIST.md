# TG/VK Parity Checklist

## Цель
Сделать так, чтобы пользователь в VK получал тот же основной функционал, что и в Telegram.

## Статус
| Функция | Telegram | VK | Статус |
|---|---|---|---|
| Идентификация пользователя | `telegram_accounts -> app_user_id` | `vk_accounts -> app_user_id` | Done |
| Единый профиль пользователя | Через `app_user_id` | Через `app_user_id` | Done |
| Привязка VK -> TG | Deep-link + token | Deep-link + token | Done |
| Просмотр профиля | `/profile`, кнопки | `Профиль` | Done |
| Просмотр списка мероприятий | `/events`, кнопки | `Мероприятия` | Done |
| Просмотр карточки мероприятия | Inline callback | `мероприятие <ID>` | Done |
| Запись на мероприятие | Inline callback + гости | `записаться <ID> [гости]` | Done |
| Отмена записи | Inline callback | `отменить <ID>` | Done |
| Список участников | В карточке | В карточке | Done |
| Фото профиля | Поддерживается | Поддерживается (вложение photo) | Done |
| Админ-команды | В Telegram | Нет в VK | Not planned |

## Что осталось
1. Добавить автотесты на паритет ключевых сценариев.
2. (Опционально) Вынести transport-слой VK/TG в отдельные модули adapters для дальнейшего масштабирования.

## Архитектурный прогресс
- Вынесен общий слой логики в `core/bot_logic.py`.
- `handlers/events.py`, `handlers/profile.py` и `vk_bot.py` используют общий `core`.
- Платформенные различия остались только в транспорте:
  - Telegram: aiogram callbacks/клавиатуры/фото.
  - VK: longpoll + текстовые команды/клавиатура VK.
